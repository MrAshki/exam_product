Start complete project in safe mock mode:
.\run-all.cmd

Start with configured AI from private root .env and mock email:
.\run-all.cmd -UseConfiguredAI

Start with configured email from private root .env and mock AI:
.\run-all.cmd -UseConfiguredEmail

Start full configured-provider mode from private root .env:
.\run-all.cmd -Clean -UseConfiguredProviders

Validate configured providers only, without starting services or calling providers:
.\run-all.cmd -CheckConfiguredProviders

Start without worker:
.\run-all.cmd -NoWorker

Clean Next cache and start:
.\run-all.cmd -Clean

Start without opening browser:
.\run-all.cmd -NoBrowser

Stop app processes:
.\stop-all.cmd

Stop app processes and Docker services:
.\stop-all.cmd -StopInfrastructure

Frontend:
http://localhost:3000

API:
http://localhost:8081

Swagger:
http://localhost:8081/docs

Always use localhost in browser-facing URLs.
Do not mix localhost and 127.0.0.1.

Default mode always forces AI_PROVIDER=mock and EMAIL_PROVIDER=mock.
Configured-provider modes read private root .env and may consume Gemini quota or send real email when you trigger AI/email actions in the app.
onboarding@resend.dev is suitable only for controlled Resend testing to the verified account recipient.
Never paste API keys, SMTP passwords, screenshots with secrets, or .env contents into terminal output, docs, Git, or chat.
